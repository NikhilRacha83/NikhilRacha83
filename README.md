## About Me

Data Engineer with **4+ years** building and owning the analytics layer of cloud data platforms, turning raw multi-source data into clean, tested, analysis-ready models that power enterprise BI and self-service reporting. Strong hands-on experience with **dbt** and **Snowflake**, advanced SQL modeling and performance tuning, and workflow orchestration with **Apache Airflow** across AWS, Azure, and GCP.

Comfortable partnering directly with analysts and business stakeholders to translate ambiguous questions into well-documented dimensional models, a governed metrics layer, and reliable data products. Healthcare analytics background spanning **member, provider, and claims data**, with a focus on data quality, governance, and delivering trustworthy datasets into **Power BI** and **Tableau**.

- Currently at **Optum Inc.** owning the Snowflake analytics layer for healthcare data
- Previously at **Infosys Ltd.** building Spark pipelines processing 200M+ records daily
- MS in Advanced Data Analytics from **University of North Texas**
- Passionate about data quality, governance, and self-serve analytics

## Work Experience

### Data Engineer | Optum Inc. *(Aug 2024 - Present)*

- Owned the Snowflake analytics layer for healthcare member, provider, and claims data, ingesting from AWS S3, Azure ADLS Gen2, and PostgreSQL, improving data availability for downstream teams by **40%**
- Designed and maintained a layered dbt project with staging, intermediate, and mart models implementing dimensional models, SCD Type 1 and 2, reusable macros, and generic/singular tests
- Partnered with stakeholders to translate ambiguous requirements into a governed metrics layer, standardizing enterprise KPI definitions so dashboards stop disagreeing on the same numbers
- Tuned Snowflake performance through virtual warehouse right-sizing, clustering key selection, and SQL refactoring, cutting query latency and monthly compute spend by **35%**
- Orchestrated ingestion, Snowflake loads, and dbt runs as Apache Airflow DAGs with retries, SLA alerting, and dbt tests wired into GitHub Actions CI/CD
- Led migration of legacy DB2 ETL to an ELT pattern on Snowflake using Fivetran connectors and Parquet COPY INTO loads

### Data Engineer | Infosys Ltd. *(Jan 2020 - Jul 2022)*

- Built and maintained Spark and PySpark batch pipelines on Azure Databricks processing **200M+ records daily**, curating large telecom usage and customer datasets
- Developed SQL and Spark ETL to load curated data into Snowflake and BigQuery, building staging and reporting tables for analysts and BI developers
- Implemented incremental loads and slowly changing dimension logic for historical customer and account data
- Optimized Spark jobs with partition pruning, broadcast joins, caching, and predicate pushdown, bringing nightly batch jobs **from hours to under an hour**
- Scheduled and monitored workflows with Apache Airflow and Apache NiFi, adding retries and alerting to reduce recurring pipeline failures
- Supported migration of legacy Hadoop ETL to cloud pipelines and early dbt/Snowflake adoption; built Tableau dashboards for telecom usage and network performance
